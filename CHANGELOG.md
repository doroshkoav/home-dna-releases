## [0.3.1] - 2026-07-08

### Добавлено
- **«Оборудование дома»** — новый экран (доступ с Дашборда и из Календаря), где можно
  отметить, что есть в квартире и в каком количестве: водонагреватель, газовый котёл,
  стиральная/посудомоечная машина, смесители, окна и двери, розетки и выключатели,
  фильтр для воды, тип напольного покрытия (ламинат/паркет/линолеум).
- **Календарь обслуживания стал data-driven**: регламентные задачи (ТО котла, замена
  картриджей смесителей, чистка стиральной/посудомоечной машины, регулировка окон и
  дверей, проверка проводки) теперь появляются и пропадают вместе с соответствующим
  оборудованием, а не показываются фиксированным списком независимо от реального
  состава квартиры. Поверка счётчиков и проверка вентиляции остаются универсальными.

### Изменено
- Расширение объёма Фазы 1 относительно исходного плана — подробности и обоснование
  решения в `plan.md` §2 репозитория `home-dna`.

### Известные ограничения
- Собрана только под arm64-v8a.
- Backend не подключён — оборудование и календарь хранятся только локально на устройстве.
- iOS-сборки нет.

[0.3.1]: https://github.com/doroshkoav/home-dna-releases/releases/tag/v0.3.1

## [0.3.0] - 2026-07-08

### Добавлено
- Вкладка **«Настройки»** (6-я в нижней навигации):
  - переключатель темы (светлая/тёмная/системная), сохраняется между запусками;
  - сброс локальных данных — для повторного прохождения онбординга при тестировании,
    пока backend не хранит данные на сервере;
  - блок «О приложении» — версия, ссылки на `home-dna` и `home-dna-releases`;
  - **проверка обновлений напрямую из GitHub Releases** — сравнивает версию с последним
    релизом `home-dna-releases`, скачивает APK и открывает системный установщик.

### Известные ограничения
- Собрана только под arm64-v8a.
- Backend не подключён — все действия работают на локальном моке, без сохранения на сервере.
- iOS-сборки нет.
- Проверка обновлений из GitHub — временный механизм только для этапа разработки. Перед
  публикацией в Google Play/App Store его нужно убрать (самообновление в обход магазина
  запрещено их правилами) — уже помечено TODO в коде.

[0.3.0]: https://github.com/doroshkoav/home-dna-releases/releases/tag/v0.3.0

## [0.2.0] - 2026-07-08

Mobile-приложение переписано с React Native на **Flutter** — решение принято сразу после
`v0.1.0`, на самом дешёвом этапе (backend ещё не подключён, мигрировать было нечего кроме
UI-кода). Причина — общее неприятие экосистемы/DX React Native, не конкретный технический
блокер. Подробности решения — `architecture.md` §11 в `home-dna`.

Функционально релиз идентичен `v0.1.0`: те же 6 экранов по тому же прототипу, тот же мок
вместо backend. Изменился только технический стек.

### Изменено
- Технология: React Native/Expo → Flutter/Dart.
- Release-сборка сразу собрана под одну архитектуру (`--target-platform android-arm64`) —
  без повторения истории с раздутым APK из `v0.1.0`.

### Известные ограничения
- Собрана только под arm64-v8a.
- Backend не подключён — все действия работают на локальном моке, без сохранения на сервере.
- iOS-сборки нет.

[0.2.0]: https://github.com/doroshkoav/home-dna-releases/releases/tag/v0.2.0

## [0.1.0] - 2026-07-08

Первая сборка — debug-APK для внутреннего теста на устройстве. Backend не подключён,
все данные локальные моки (см. `apps/mobile/src/context/AppState.tsx` в `home-dna`).

### Добавлено
- 6 экранов мобильного приложения по прототипу (`design/prototype`): Онбординг, Дашборд,
  Документы, Гарантии, Журнал, Календарь — с состояниями пусто/загрузка/заполнено/ошибка.
- Тёплая цветовая тема (светлая/тёмная) на основе `packages/design-tokens`.

### Известные ограничения
- Debug-сборка, не подписана для публикации в маркеты.
- Собрана только под arm64-v8a (наиболее распространённая архитектура) — armeabi-v7a/x86/x86_64 не включены в этот билд.
- Backend не подключён — все действия (загрузка документов, гарантий и т.д.) работают на локальном моке, без сохранения на сервере.
- iOS-сборки нет.

### Исправлено
- Размер APK уменьшен со 132 МБ до ~44 МБ: `-PreactNativeArchitectures` ограничивал только компиляцию
  собственного нативного кода, но не упаковку прекомпилированных `.so` из React Native/Hermes — добавлен
  `android.defaultConfig.ndk.abiFilters "arm64-v8a"` в `apps/mobile/android/app/build.gradle`, чтобы
  реально исключить лишние архитектуры из финального APK.
