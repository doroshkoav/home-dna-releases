## [0.4.10] - 2026-07-10

### Добавлено
- **Свой ЖЦ (жизненный цикл) для каждого компонента оборудования** — у оборудования
  теперь может быть несколько узлов с разной периодичностью ТО (например, у смесителя:
  картридж раз в год, гибкий шланг раз в 5 лет, фильтр грубой очистки раз в 3 месяца).
  Компоненты заводятся автоматически по типу оборудования, можно добавлять свои и
  удалять — история ТО сохраняет название удалённого компонента.
- Календарь обслуживания объединяет универсальные задачи (поверка счётчиков, чистка
  вентиляции) и регламенты компонентов оборудования в один список с разбивкой «в
  ближайший месяц» / «позже».

### Известные ограничения
- Собрана только под arm64-v8a.
- Соединение с сервером пока без HTTPS.
- iOS-сборки нет.

[0.4.10]: https://github.com/doroshkoav/home-dna-releases/releases/tag/v0.4.10

## [0.4.9] - 2026-07-09

  показывается уведомление со следующей датой ТО, а кнопка на время запроса — со
  спиннером.

### Известные ограничения

## [0.4.8] - 2026-07-09

### Исправлено
- **Белый экран при выборе даты покупки/установки** (и на карточке оборудования, и при
  добавлении нового) — приложению не хватало пакета локализации для диалога выбора даты
  на русском, из-за чего он падал при открытии.
- **«История ТО» на карточке не обновлялась** после добавления записи вручную и
  возврата с экрана истории — превью показывало старый снимок («записей пока нет»),
  хотя запись уже была сохранена.
- Переименование карточки и изменение даты установки теперь сразу отражаются на самой
  карточке, а не только после повторного захода.

### Известные ограничения
- Собрана только под arm64-v8a.
- Соединение с сервером пока без HTTPS.
- iOS-сборки нет.

## [0.4.6] - 2026-07-09

### Исправлено
- **Приложение зависало на экране загрузки после обновления с версии до 0.4.5** —
  локальный офлайн-кэш, сохранённый старой версией, был в формате оборудования

## [0.4.5] - 2026-07-09

### Добавлено
- **Оборудование теперь карточки, а не просто счётчик количества**: у каждой единицы —
  своё название и дата покупки/установки. На карточке — превью «История ТО», которое
  открывает полноценный таймлайн: и плановые записи (сервер сам добавляет их при
  завершении регламентной задачи в календаре), и внеплановые — с мастером, стоимостью и
  фото, добавляются вручную.
- Регламентная задача ТО теперь привязана к конкретной единице оборудования (а не к
  типу на весь объект) и по-настоящему повторяется: отметка «выполнено» сдвигает её на
  следующий период вместо того, чтобы просто погасить навсегда.
- Экран «Календарь» показывает, к какому оборудованию относится задача.

### Изменено
- Добавление оборудования — через мини-форму (название, дата установки) вместо
  степпера количества; каждая единица добавляется отдельно.

### Известные ограничения
- Собрана только под arm64-v8a.
- Соединение с сервером пока без HTTPS.
- iOS-сборки нет.

## [0.4.0] - 2026-07-09

  недвижимости, документы, гарантии, журнал, календарь, оборудование) теперь

## [0.2.0] - 2026-07-08

Mobile-приложение переписано с React Native на **Flutter** — решение принято сразу после
`v0.1.0`, на самом дешёвом этапе (backend ещё не подключён, мигрировать было нечего кроме
UI-кода). Причина — общее неприятие экосистемы/DX React Native, не конкретный технический
блокер. Подробности решения — `architecture.md` §11 в `home-dna`.

Функционально релиз идентичен `v0.1.0`: те же 6 экранов по тому же прототипу, тот же мок
вместо backend. Изменился только технический стек.

### Изменено
- Технология: React Native/Expo → Flutter/Dart.
- Release-сборка сразу собрана под одну архитектуру (`--target-platform android-arm64`) —
  без повторения истории с раздутым APK из `v0.1.0`.

### Известные ограничения
- Собрана только под arm64-v8a.
- Backend не подключён — все действия работают на локальном моке, без сохранения на сервере.
- iOS-сборки нет.

[0.2.0]: https://github.com/doroshkoav/home-dna-releases/releases/tag/v0.2.0

## [0.1.0] - 2026-07-08

Первая сборка — debug-APK для внутреннего теста на устройстве. Backend не подключён,
все данные локальные моки (см. `apps/mobile/src/context/AppState.tsx` в `home-dna`).

### Добавлено
- 6 экранов мобильного приложения по прототипу (`design/prototype`): Онбординг, Дашборд,
  Документы, Гарантии, Журнал, Календарь — с состояниями пусто/загрузка/заполнено/ошибка.
- Тёплая цветовая тема (светлая/тёмная) на основе `packages/design-tokens`.

### Известные ограничения
- Debug-сборка, не подписана для публикации в маркеты.
- Собрана только под arm64-v8a (наиболее распространённая архитектура) — armeabi-v7a/x86/x86_64 не включены в этот билд.
- Backend не подключён — все действия (загрузка документов, гарантий и т.д.) работают на локальном моке, без сохранения на сервере.
- iOS-сборки нет.

### Исправлено
- Размер APK уменьшен со 132 МБ до ~44 МБ: `-PreactNativeArchitectures` ограничивал только компиляцию
  собственного нативного кода, но не упаковку прекомпилированных `.so` из React Native/Hermes — добавлен
  `android.defaultConfig.ndk.abiFilters "arm64-v8a"` в `apps/mobile/android/app/build.gradle`, чтобы
  реально исключить лишние архитектуры из финального APK.
